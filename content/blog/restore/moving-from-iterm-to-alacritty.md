A few years ago I [decided](/starting-vim-journey-again/) to use Vim as my text editor. It has been a fantastic journey so far. I have learned a lot about Vim and I keep learning new things every day. This continuous learning makes the tool so interesting. For coding, I also use Visual Studio Code which I think is a fantastic _almost IDE_. However, most of the work I do on a daily bases is in Vim. My setup is iTerm, Tmux and Vim. This trio works well for me and because I have most of the shortcuts ingrained into my muscle memory. Since last few months, I felt that while Tmux and Vim were living up to their promise, iTerm at times felt a bit sluggish. Once in a while, I felt that it could be faster. To remedy the situation, I did what anyone would do. I googled how to make iTerm faster. While browsing through various Reddits and Stackoverlow, I found out about another terminal emulator called [Alacritty](https://github.com/jwilm/alacritty) which claims to be the fastest terminal emulators out there. I decided to give it a try for a few days and I came to a point where I ditched iTerm and adapted Alacritty as my main terminal app. Alacritty is fast. It is fast enough for you to notice the difference in speed immediately. The reason for the speed is that Alacritty uses the GPU for rendering. It feels snappy and there is no janking. Setting up Alacritty is a breeze. As I use a mac, I used brew to install it. I like my terminal, and editor to look a certain way. The color scheme is important to me. On iTerm I have been using the [Ayu](https://github.com/ayu-theme/ayu-colors) color scheme and it was important to me to have it on Alacritty. If Ayu was not available for Alacritty, then my plan was to create it. Luckily I found the colors in the [wiki](https://github.com/jwilm/alacritty/wiki/Color-schemes) and it was just a matter of updating the config file. The config for Alacritty lives in .config/alacritty/alacritty.yml The file is well commented and you can find most of the settings by just reading the comments. Alacritty does have a few limitations. The biggest one is that there is no support for split terminals. This is not an issue for me because I use Tmux for splitting panes within a Tmux window. This is what Alacritty looks like running Tmux and Vim on Ayu color scheme. ![](https://deekap.com/wp-content/uploads/2019/07/alacritty_tmux_vim-1024x813.png) Today is day 4 of me using Alacritty and so far I have not come across any reason to abandon it.
